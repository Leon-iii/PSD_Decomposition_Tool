0. 개요

이 문서는 PSD 파일을 집어넣으면 레이어를 감지, 특정 레이어 또는 모든 레이어를 별도의 PSD or PNG 파일으로 export하는 프로그램의 사양 설명서이다.

이 프로그램은 다음과 같은 구조로 이루어진다.

- 파일 입력 계층
- 레이어 선택 계층
- 출력 설정 계층
- 처리 계층

1. 파일 입력 계층

처리할 파일을 프로그램에 업로드한다. 파일 찾기 버튼, 드래그 앤 드롭을 지원해야 한다.
현재는 PSD만 지원하지만, 나중에 .ase 등 다른 확장자를 추가하는 등 확장의 여지를 열어 둬야 한다.
지원되지 않는 확장자의 파일을 드래그 앤 드롭 시도하는 경우 경고 문구를 표시해야 한다.

2. 레이어 선택 계층

파일의 모든 레이어를 불러와서 화면에 표시한다.
체크박스 형태로 이루어진 별도 UI가 존재해야 한다.
예:
□ background
□ building

3. 출력 설정 계층

선택된 레이어를 어떻게 출력할 지 configure하는 계층이다.
프로그램 종료 후에 로드 시에도 저장된 값을 유지할 수 있도록 직렬화/역직렬화 시스템을 사용해야 한다.
포함해야 할 설정 사항은 다음과 같다.

Output Directory: [Path] [경로 찾기...]
□ Wrap with folder
File Name: □ Original File Name □ Layer name □ Date
Export Format: □ PSD / □ PNG
Rescale: □ 100% □ 200% □ 400% □ 800%

Output Directory: 파일이 저장될 위치를 지정한다. 기본값은 원본 파일 위치이다.

Wrap with folder = True이면 출력 파일들을 폴더로 감싼다. 폴더명을 "(원본 파일 이름)_decomposed" 로 한다.

File Name: 파일 이름을 결정한다.
- Original File Name = True이면 출력 파일 이름에 원본 파일 이름을 추가한다.
- Layer Name = True이면 출력 파일 이름에 해당 레이어 이름을 추가한다.
- Date = True이면 출력 파일 이름에 현재 날짜 (예: 260507)을 추가한다.
- 여러 가지 항목이 True인 경우 언더스코어를 가운데에 넣어 Concatenate한다. (예: City1_sky_260507)
- 같은 이름의 파일이 여럿일 경우 자동으로 숫자를 추가해 오류를 회피한다. (예: City1_260507_2)

Export Format: 저장할 파일의 확장자를 결정한다.
PSD 모드에서는 export할 레이어 수 만큼 파일을 복제한 다음, 모든 파일을 열어 보존할 레이어 이외의 레이어를 삭제하는 방식으로 우회한다.
PNG 모드에서는 그렇게 할 필요가 없으니 그냥 각 레이어를 렌더링해 저장하는 방식을 택한다.

Rescale: 저장 시에 파일을 확대한다.

4. 처리 계층

입력된 정보를 바탕으로 실제 처리를 수행한다. 구현은 자율적으로 진행하며, 흐름 추적 및 유지보수가 쉽도록 각 계층을 모듈 단위로 분해하여 수행한다.

5. 주의사항

이 프로그램은 tkinter, ttk를 사용한 GUI 프로그램이어야 한다.
